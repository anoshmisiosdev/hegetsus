PHP Database

The PHP database is the keystone of our website. It stores data in tables that can be accessed through queries that are automatically generated by the various pages PHP code, allowing them to be updated and edited without having to individually change the code of the pages.
The website also features a custom password system. The password is stored in the database, and is compared to the user input to determine whether they get access to the editor. The password page utilizes redirects to an action php page that performs the processing, and returns the user to the login page if the password is incorrect.
The editor utilizes an input form that sends its data to another php action page which generates an SQL query to edit the stored data for the website. One of the main issues we had was that the editor was accessible both through typing the correct password and through typing the editor URL directly into the adress bar. We prevented this by using a check at the beginning of the editor page to check for the presence of a tracer that could only be sent by the password redirect page, which grealty improved the security of the editor.
